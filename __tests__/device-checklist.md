# 実機テスト チェックリスト

対象URL: https://reachapp.jp  
テスト端末: スマートフォン（iOS / Android）+ PC ブラウザ

---

## 認証

- [ ] メールアドレス + パスワードでログインできる
- [ ] ログアウトできる
- [ ] ログアウト後にトーク画面にアクセスするとログイン画面にリダイレクトされる

## トーク一覧（talk）

- [ ] フォロー中クリエイターの最新配信が表示される
- [ ] 未読バッジが正しく表示される
- [ ] 配信をタップすると詳細画面に遷移する
- [ ] DM セクションが表示される
- [ ] スワイプでピン止め・通知オフ・削除ができる（モバイル）

## 配信詳細（talk/[id]）

- [ ] テキスト配信が正しく表示される
- [ ] 画像配信が表示される
- [ ] 動画配信が再生コントロール付きで表示される ← 新機能
- [ ] いいねボタンが動作する
- [ ] DM 入力欄から送信できる
- [ ] タイルメニューが表示・タップできる（設定済みの場合）
- [ ] 鍵アカウントで未承認フォロワーにはコンテンツが見えない

## 配信作成（compose）

- [ ] テキスト入力して配信できる
- [ ] 画像を選択・アップロードして配信できる
- [ ] 動画を選択・アップロードして配信できる ← 新機能
- [ ] 下書き保存できる
- [ ] 予約配信が設定できる
- [ ] MB限定配信が設定できる（メンバーシップ設定済みの場合）

## フロー配信

- [ ] フォロー直後に day_offset=0 のメッセージが届く
- [ ] step-sequences 一覧が表示される
- [ ] シーケンス編集・保存できる
- [ ] 翌日以降のメッセージが cron で送信される（手動確認）

## クリエイタープロフィール

- [ ] フォローボタンが動作する
- [ ] 二重タップしてもフォローリクエストが1件だけ作成される ← Fix#4
- [ ] 鍵垢クリエイターにフォローリクエストが送れる
- [ ] メンバーシップボタンが表示される（membership_active=true の場合）

## メンバーシップ

- [ ] 公開クリエイターのメンバーシップページが見える
- [ ] 鍵垢クリエイターで未承認フォロワーがアクセスするとガード表示 ← Fix#6
- [ ] 加入フロー（Stripe）が動作する

## 設定

- [ ] 鍵アカウント切り替えが保存される
- [ ] 失敗時にエラーダイアログが表示される ← Fix#1
- [ ] プッシュ通知切り替えが保存される

## パフォーマンス（体感）

- [ ] 初回ロードが 5 秒以内
- [ ] ページ遷移が 2 秒以内
- [ ] 動画のバッファリングが始まる

## レイアウト

- [ ] モバイル（375px）: 崩れがない
- [ ] タブレット（768px）: 崩れがない
- [ ] デスクトップ（1280px）: 2カラムレイアウトが表示される
